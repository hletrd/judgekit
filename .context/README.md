# Repository Context

This directory captures high-signal project context for future work sessions.

- `project/current-state.md` tracks the latest shipped and deployed state that docs should reflect.
- `development/open-workstreams.md` tracks unfinished local work so future sessions do not describe or ship it as complete.
- `AGENTS.md` remains the primary repo-level instruction file; use these notes as a companion, not a replacement.
